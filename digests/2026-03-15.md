## Key Changes

- **[Bug Fix] Pipeline Parallelism race condition fixed (#20341)**
  `scheduler_pp_mixin.py`: proxy tensors and output tensors sent between PP stages could be consumed out of order under concurrency. Fix tags every outgoing tensor dict with a `__msg_type__` key (`"proxy"` or `"output"`) and demultiplexes incoming messages into a per-type inbox queue (`_pp_tensor_dict_inbox`). Directly affects multi-node pipeline-parallel inference stability.

- **[New Model] Nemotron 3 Super NVFP4 / mixed-precision quantization (#20407)**
  Adds a new `modelopt_mixed` quantization method for NVIDIA's Nemotron 3 Super model, which uses a per-layer mix of NVFP4 and FP8 weights. Introduces `ModelOptMixedPrecisionConfig` in `modelopt_quant.py` and detection logic in `model_config.py`. Ships with `test_modelopt_loader.py`. Only supported via pre-quantized checkpoints (dynamic quant raises `ValueError`).

- **[Kernel Slimming] Remove deprecated AOT CUDA kernels from sgl-kernel (#20277)**
  Deletes ~1,483 lines across `csrc/elementwise/rope.cu`, `csrc/gemm/per_tensor_quant_fp8.cu`, and `csrc/memory/store.cu`, along with their Python wrappers and 3 test files (`test_rotary_embedding.py`, `test_fp4_quantize.py`, `test_per_tensor_quant_fp8.py`). Benchmark callers migrated to JIT kernel equivalents (e.g. `sglang.jit_kernel.nvfp4`). Net −1,423 lines.

- **[CI] Ascend NPU GPTQ tests migrated to `test/registered` (#19628)**
  Moves Ascend GPTQ tests out of the legacy `test/srt` path, adds a dedicated `test_ascend_qwen3_1_7b_gptq_int8.py`, and trims `test_ascend_utils.py` by ~65%. Updates `.github/workflows/pr-test-npu.yml` to register the new suite.

- **[Diffusion] MoVA and HunyuanVideo added to perf benchmark scripts (#20563)**
  Extends `bench_diffusion_denoise.py` with model configs for MoVA and HunyuanVideo; updates the corresponding skills doc. Tooling/docs only — no runtime impact.

## Watchlist

1. **PP race condition fix — regression risk** (#20341)
   The new typed inbox (`_pp_tensor_dict_inbox`) and `msg_type` tagging are non-trivial changes to the PP message loop. Any caller invoking `_pp_send_dict_to_next_stage` without an explicit `msg_type` now logs a warning but continues — silent behavior change. Multi-stage PP end-to-end tests should be monitored on next CI runs.

2. **Deprecated kernel removal may break un-migrated callers** (#20277)
   Three AOT kernels (RoPE, per-tensor FP8 quant, KV store) are now gone. Any external integration or downstream code that imported these directly from `sgl_kernel` (e.g. `from sgl_kernel import scaled_fp4_quant`) will fail at import time. The JIT replacements are functionally equivalent but untested in all the same configurations.

3. **`modelopt_mixed` quant — limited validation** (#20407)
   New quantization path has a single loader test but no inference-level integration test. The `ValueError` guard for dynamic quantization could mask misconfigured environments. Worth adding a smoke test against an actual Nemotron 3 Super checkpoint before declaring production-ready.
